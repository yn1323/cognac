# タスク詳細表示（Overview タブ）実装方針

## 概要

Task画面のOverviewタブで、DBに保存されたタスクの詳細を表示する。
Selected Personas と Task Progress はモックデータを維持。編集・削除機能は対象外。

## デザイン参照

- Pencil Node ID: `4UNiS`（Cognac - Task Overview 画面全体）
- PC版 Overview: `9d5bz`
- SP版 Overview: `lNPXJ`

## 変更対象ファイル

| ファイル | 変更内容 |
|---------|---------|
| `client/lib/format.ts` | `formatDateTime()` を追加（null対応、YYYY/M/D H:MM形式） |
| `client/lib/status-config.ts` | `STATUS_PHASE_MAP` を追加（ステータス→フェーズ名マッピング） |
| `client/pages/task-page.tsx` | `useParams` + `useTask()` でAPI接続、loading/error状態、子へtask prop渡し |
| `client/pages/task-detail/overview-tab.tsx` | `task` propを受け取りTask Informationカードに実データ表示 |
| `client/components/task-detail/task-detail.tsx` | ローカル `formatDateTime` を `@/lib/format` からの import に変更 |

## アーキテクチャ決定

### データ取得戦略

`TaskPage`（ルートコンポーネント）で `useParams` + `useTask(id)` を呼び出し、`task` を props で子コンポーネントに渡す。`DashboardPage` と同じパターン。

### ステータス → フェーズ名マッピング

```
pending    → Queued
discussing → Phase 2-B: Multi-Persona Discussion
planned    → Phase 2-C: Plan Confirmed
executing  → Phase 3: Code Execution
testing    → Phase 4: CI Testing
completed  → Completed
paused     → Paused（task.paused_phase があれば優先表示）
stopped    → Stopped
```

## Task Information カードの表示フィールド

| UI表示 | データソース | 備考 |
|--------|------------|------|
| Status | `task.status` | Badge variant で色分け |
| Current Phase | `STATUS_PHASE_MAP[task.status]` | paused時は `task.paused_phase` 優先 |
| Branch | `task.branch_name` | null なら `-` |
| Created | `task.created_at` | `formatDateTime()` で整形 |
| Description | `task.description` | null なら `No description` |

## Loading / Error 状態

- **Loading**: 全画面中央に `Loader2` スピナー
- **Error / Not Found**: `AlertCircle` + メッセージ + ダッシュボードへの戻るボタン
