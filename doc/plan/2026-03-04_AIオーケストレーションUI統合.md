# AIオーケストレーション UI統合計画

## Context

サーバー側のAIオーケストレーションパイプライン（Phase 2-A ペルソナ選定 → 2-B ディスカッション → 2-C プラン策定 → Phase 3 実行 → CI）は実装済み。SSEイベントもDB永続化も動作する。

しかし、クライアントのタスク詳細ページ（概要・ディスカッション・プラン・CIタブ）は**全てモックデータ**で表示されており、実データと繋がっていない。SSE接続もログタブでしか有効にならない。

**ゴール:** 各タブをサーバーの実データに接続し、パイプライン実行中はSSEでリアルタイム更新されるようにする。

---

## 実装ステップ

### Step 1: サーバーAPIエンドポイント追加 — [server/api/tasks.ts](server/api/tasks.ts)

3つのGETエンドポイントを追加。既存パターン（`GET /:id/logs`, `GET /:id/images`）に準拠。

```typescript
import * as personaQueries from '../db/queries/personas.js'
import * as discussionQueries from '../db/queries/discussions.js'
import * as planQueries from '../db/queries/plans.js'
```

| エンドポイント | レスポンス | 備考 |
|---|---|---|
| `GET /:id/personas` | `Persona[]` | 空配列=まだ選定されてない |
| `GET /:id/discussions` | `Discussion[]` | 空配列=まだディスカッションなし |
| `GET /:id/plan` | `Plan \| null` | `null`=まだプランなし（404ではない） |

全て、タスク自体が存在しない場合のみ404を返す。

```typescript
getPersonas: (taskId: number) => fetchJson<Persona[]>(`/tasks/${taskId}/personas`),
getDiscussions: (taskId: number) => fetchJson<Discussion[]>(`/tasks/${taskId}/discussions`),

```

### Step 4: ペルソナカラーユーティリティ — `client/lib/persona-colors.ts`（新規）

概要タブとディスカッションタブで共有する、ペルソナIDから色を決定するユーティリティ。

```typescript
const PERSONA_COLORS = ['#7c3aed', '#2563eb', '#ea580c', '#16a34a', '#db2777', '#0891b2']

export function getPersonaColor(index: number): string {
  return PERSONA_COLORS[index % PERSONA_COLORS.length]
}
```

インデックスベース（タスク内のペルソナ配列の位置で決定）。

### Step 5: SSE接続のページ全体化 + キャッシュ無効化 — [client/pages/task-page.tsx](client/pages/task-page.tsx)

**現状の問題:** SSEは `activeTab === 'ログ'` の時だけ接続される（L348）。他のタブではリアルタイム更新されない。

**変更内容:**

1. `shouldConnectSSE` から `activeTab === 'ログ'` 条件を削除:
   ```typescript
   const shouldConnectSSE = task != null && ACTIVE_STATUSES.has(task.status)
   ```

2. SSEイベントに応じてReact Queryキャッシュを無効化する `useEffect` を追加:
   ```typescript
   useEffect(() => {
     if (sseEvents.length === 0 || !task) return
     const latest = sseEvents[sseEvents.length - 1]
     if (latest.type === 'persona_selected') {
       qc.invalidateQueries({ queryKey: ['tasks', task.id, 'personas'] })
     }
     if (['discussion_round_end', 'discussion_statement'].includes(latest.type)) {
       qc.invalidateQueries({ queryKey: ['tasks', task.id, 'discussions'] })
     }
     if (latest.type === 'plan_created') {
       qc.invalidateQueries({ queryKey: ['tasks', task.id, 'plan'] })
     }
   }, [sseEvents.length])
   ```

3. 各タブコンポーネントに `task` propを渡す（ディスカッション、プラン、CI）:
   ```typescript
   case 'ディスカッション':
     return <PCDiscussionTab task={task} />
   case 'プラン':
     return <PCPlanTab task={task} />
   case 'CI':
     return <PCCITab task={task} events={sseEvents} />
   ```

**アーキテクチャ方針:** RESTが信頼できるデータソース、SSEはキャッシュ無効化のトリガー。SSEイベントから直接UIステートを構築しない。

---

### Step 6: 概要タブ — [client/pages/task-detail/overview-tab.tsx](client/pages/task-detail/overview-tab.tsx)

**変更内容:**

1. `MOCK_PERSONAS` と `MOCK_PROGRESS` を削除
2. `useTaskPersonas(task.id)` でペルソナ取得、`getPersonaColor()` で色を決定
3. タスク進捗をタスクステータス + 取得データから動的に計算:
   - `pending`: 全ステップ `waiting`
   - `discussing`: Phase 2-A = `completed`、2-B = `active`（ペルソナ数とラウンド数をsub表示）
   - `planned`: 2-A/2-B/2-C = `completed`、Phase 3 = `waiting`
   - `executing`: 2-A/2-B/2-C = `completed`、Phase 3 = `active`
   - `testing`: Phase 3 = `completed`、CI = `active`
   - `completed`: 全ステップ `completed`
   - `paused`/`stopped`: `paused_phase` に基づいて表示
4. ペルソナ/進捗が空の場合の空状態メッセージ: 「タスクの実行待ち」

---

### Step 7: ディスカッションタブ — [client/pages/task-detail/discussion-tab.tsx](client/pages/task-detail/discussion-tab.tsx)

**変更内容:**

1. `MOCK_MESSAGES` を削除
2. `PCDiscussionTab({ task })` / `SPDiscussionTab({ task })` にシグネチャ変更
3. `useTaskDiscussions(task.id)` でディスカッション取得
4. `useMemo` でラウンドごとにグルーピング（`Map<number, Discussion[]>`）
5. 各ラウンドのセパレーターを表示 → ラウンド内の発言をチャットバブルで描画
6. `key_points` を `JSON.parse()` してバッジ/タグとして表示
7. ペルソナの色は `useTaskPersonas` で取得したペルソナ配列のインデックスで決定
8. 空状態: 「ディスカッション開始を待っています」
9. 最終ラウンドの `should_continue === false` のとき「合意形成」マーカー表示

---

### Step 8: プランタブ — [client/pages/task-detail/plan-tab.tsx](client/pages/task-detail/plan-tab.tsx)

**変更内容:**

1. `MOCK_PLAN_STEPS` と `MOCK_PROMPT_LINES` を削除
2. `PCPlanTab({ task })` / `SPPlanTab({ task })` にシグネチャ変更
3. `useTaskPlan(task.id)` でプラン取得
4. `plan_markdown` を `<pre className="whitespace-pre-wrap">` でフォーマット表示（react-markdown依存なし）
5. `execution_prompt` を既存のダークコードブロックスタイルで表示
6. `estimated_complexity` をBadgeで表示（low=緑、medium=黄、high=赤）
7. コピーボタンに `navigator.clipboard.writeText()` を実装
8. 空状態: 「プラン生成を待っています」

---

### Step 9: CIタブ — [client/pages/task-detail/ci-tab.tsx](client/pages/task-detail/ci-tab.tsx)

**変更内容:**

1. ハードコードされた値を削除
2. `PCCITab({ task, events })` / `SPCITab({ task, events })` にシグネチャ変更
3. アクティブタスク: SSEイベントから `ci_start`/`ci_result` をフィルタして表示
   - 各CIステップ（typecheck, lint, test, build）の成功/失敗/進行中を表示
   - `task.retry_count` でリトライ回数表示
4. 完了/停止タスク: `useTaskLogs(task.id)` から `phase === 'ci'` のログを取得、`output_summary` を表示
5. ステータスバナーの色を動的に:
   - 進行中: 青
   - 全成功: 緑
   - 失敗あり: 赤
6. 空状態: 「CI実行を待っています」

## 実装順序

```
Step 1 (サーバーAPI)  ─┐
Step 2 (APIクライアント) ─┤
Step 3 (Queryフック)     ─┼→ Step 5 (SSE) → Step 6〜9 (各タブ)
Step 4 (ペルソナカラー)   ─┘
```

Steps 1-4は独立しているため先に片付ける。Step 5のSSE変更後、各タブを順に更新。
