# タスク作成機能 - 実装方針

## Context

TaskModal（タスク作成ダイアログ）のUIは既に実装済みだったが、`handleSubmit`が`console.log`のスタブ状態だった。
`useCreateTask` hookやAPI clientも存在したが、以下が未対応だった:
- `CreateTaskInput`型に`priority`フィールドがない
- 画像アップロードのバックエンドが未実装（`task_images`テーブルは既存）
- TaskModalのsubmitがAPIに接続されていない

**目的**: TaskModalからタスクを作成し（title, description, priority, images）、APIを通じてSQLite DBに保存できる状態にする。

---

## 実装方針

### タスク作成フロー（2ステップ）

1. `POST /api/tasks` (JSON) → タスク作成 → task IDを取得
2. `POST /api/tasks/:id/images` (multipart/form-data) → 画像をアップロード

クライアント側では「Create Task」ボタン1回で両方を順番に実行する。

### 画像保存先

`.cognac/uploads/<taskId>/` ディレクトリにファイルを保存。
DBの`task_images`テーブルにメタデータを記録。

## 画像アップロード仕様

- Honoの `c.req.formData()` でmultipart解析
- バリデーション: image/* のみ、1ファイル5MB上限、最大5ファイル
- ファイル名は `randomUUID() + 拡張子` で保存（衝突回避）
- レスポンス: 作成された `TaskImage[]` を返す

## クライアント画像UI

- `DropZone` コンポーネント: ドラッグ&ドロップ + クリックでファイル選択
- `ImagePreviewList` コンポーネント: `URL.createObjectURL` でプレビュー + 個別削除
- PC版: `Upload` アイコン + テキスト / SP版: `Camera` アイコン + テキスト

## 変更しないもの

- DBスキーマ（`task_images`テーブルは既存）
- ルーティング（TaskModalは既にダッシュボードに組み込み済み）
- SSE / TaskRunner（タスク作成後の自動実行は既存機能）
